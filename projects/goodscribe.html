Engineering Challenges

Ultra-Low Latency Audio

Problem

Standard browser audio processing introduces delays (2s+) unacceptable for live interpretation.

Solution

Implemented custom AudioWorklets to process raw PCM audio on a separate thread, bypassing the main event loop and streaming directly via WebSockets.

Serverless Security Architecture

Problem

Directly accessing Azure & Google Cloud APIs from the client would expose sensitive billing keys.

Solution

Implemented a reverse proxy using Firebase Cloud Functions and Cloudflare Workers. The client authenticates via Firebase Auth, and the server injects the secrets before calling upstream APIs.

Extension State Synchronization

Problem

Managing state across three isolated contexts: Popup, Background Worker, and Content Script.

Solution

Built a reactive message-passing architecture using Chrome's Runtime API to ensure UI updates instantly reflect background process status.

Real-time Cost Guardrails

Problem

Preventing API bill shock from users exceeding their plan limits on pay-per-use AI services.

Solution

Engineered atomic pre-flight checks in Firestore. Every API request validates the user's "minutes used" against their plan limits before processing audio.
